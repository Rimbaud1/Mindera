1 Contexte et Objectifs du Projet

Nom du projet

Webstra Prospector

Objectif principal

Développer un backend Python intelligent couplé à une interface web simple pour automatiser la prospection de clients (artisans, restaurateurs, petites entreprises) pour l'agence Webstra.

Caractéristiques clés

  • Backend Python intelligent
  • Interface web simple
  • Utilisation locale uniquement
  • Automatisation complète du processus

2 Fonctionnalités Clés

Backend Python (API ou Script Local)

Recherche de prospects

Par zone géographique (API Google Maps, Pages Jaunes, ou scraping simple)

Analyse de présence web

  • Site existant ou non
  • Responsive ou non
  • Design moderne
  • Ancienneté domaine
  • Référencement SEO

Classement des leads

Par priorité (score global calculé automatiquement)

Génération de pitchs

Personnalisés avec OpenAI (ou autre LLM)

Envoi d'emails automatisés

Via SMTP avec tracking (Pixel invisible)

Stockage local

Base de données SQLite pour les leads

Frontend Web (Interface Simple en Local)

Interface locale sécurisée

  • Fonctionne uniquement en local
  • Sans authentification complexe
  • Sélection de zone géographique (ville/département)

Tableau de bord

  • Classement des leads par priorité
  • Détails complets pour chaque prospect
  • Génération de mails personnalisés
  • Historique des actions
