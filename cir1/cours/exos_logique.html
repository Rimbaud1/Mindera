Exercices de Syllogisme

Entraînement aux raisonnements conditionnels, disjonctifs et d'incompatibilité

Comment travailler avec ces exercices

  1. Identifiez le type de majeure : implication (Si p alors q), disjonction (p ou q) ou incompatibilité (non(p et q)).
  2. Repérez la mineure : affirmation ou négation de quel terme ?
  3. Appliquez la forme canonique (Modus Ponens/Tollens, Ponendo Tollens, Tollendo Ponens).
  4. Vérifiez la validité : certaines formes ne fonctionnent que pour des majores strictes ou avec des mineures affirmatives.
  5. Rédigez clairement la conclusion ; si aucune ne s'impose, indiquez "Aucune conclusion certaine".

Syllogismes Conditionnels

Forme de base : Si p alors q

Exercice C-1

Énoncé : Si je révise régulièrement (p), je réussis mon partiel (q).

Prémisse mineure : Or je révise régulièrement (p).

Que puis-je conclure ?

Solution : Je réussis mon partiel (q).

Explication : Modus Ponens - on affirme l'antécédent (p), donc on peut affirmer le conséquent (q). Raisonnement valide.

Exercice C-2

Énoncé : Si un composant surchauffe (p), alors le système s'éteint (q).

Prémisse mineure : Or le système ne s'est pas éteint (¬q).

Que dire sur p ?

Solution : Le composant n'a pas surchauffé (¬p).

Explication : Modus Tollens - on nie le conséquent (q), donc on peut nier l'antécédent (p). Raisonnement valide.

Exercice C-3

Énoncé : Si le programme compile (p) alors il est exempt d'erreurs de syntaxe (q).

Prémisse mineure : Or le programme compile (p).

Peut-on en déduire qu'il est optimisé (r) ?

Solution : Aucune conclusion.

Explication : L'optimisation (r) n'apparaît nulle part dans les prémisses. On ne peut rien en déduire sur ce point.

Exercice C-4

Énoncé : Si un nombre est multiple de 4 (p) alors il est pair (q).

Prémisse mineure : Or le nombre est pair (q).

Peut-on conclure qu'il est multiple de 4 (p) ?

Solution : Impossible.

Explication : Affirmation du conséquent - sophisme classique. Un nombre pair (q) peut être multiple de 4 (p) ou non (ex: 6 est pair mais pas multiple de 4).

Exercice C-5

Énoncé : Si tu es admissible (p) alors tu passes l'orale (q).

Prémisse mineure : Tu n'es pas admissible (¬p).

Peut-on conclure que tu ne passes pas l'orale (¬q) ?

Solution : Aucune conclusion.

Explication : Négation de l'antécédent - sophisme. Ne pas être admissible (¬p) ne renseigne pas sur q (on pourrait passer l'orale par d'autres moyens).

Syllogismes Disjonctifs

Forme de base : p ou q (préciser si disjonction stricte ou large)

Exercice D-1 Stricte

Énoncé : Le serveur est soit en maintenance (p) soit hors-ligne pour bug (q).

Prémisse mineure : Or il est en maintenance (p).

Conclusion ?

Solution : Il n'est pas hors-ligne (¬q).

Explication : Ponendo Tollens valide car disjonction stricte (XOR). Si p est vrai alors q doit être faux (et vice versa).

Exercice D-2 Large

Énoncé : Le code est documenté (p) ou testé (q).

Prémisse mineure : Or il n'est pas documenté (¬p).

Conclusion ?

Solution : Il est testé (q).

Explication : Tollendo Ponens valide pour les deux types de disjonctions. Si p est faux alors q doit être vrai (dans ce cas où la disjonction est large).

Exercice D-3 Stricte

Énoncé : La LED est rouge (p) ou verte (q).

Prémisse mineure : Or elle n'est pas rouge (¬p).

Conclusion ?

Solution : Verte (q).

Explication : Tollendo Ponens, valide. Pour une disjonction stricte, si p est faux alors q doit être vrai (et vice versa).

Exercice D-4 Large

Énoncé : Il pleut (p) ou il vente (q).

Prémisse mineure : Or il pleut (p).

Que conclure sur q ?

Solution : Aucune certitude.

Explication : Ponendo Tollens invalide en disjonction large. Si p est vrai, q peut être vrai ou faux (il peut pleuvoir et venter en même temps).

Exercice D-5 Stricte

Énoncé : Le fichier est soit texte (p) soit binaire (q).

Prémisse mineure : Or il est binaire (q).

Conclusion sur p ?

Solution : Il n'est pas texte (¬p).

Explication : Symétrique de D-1. Pour une disjonction stricte, si q est vrai alors p doit être faux.

Syllogismes d'Incompatibilité

Forme de base : non(p et q)

Exercice I-1

Énoncé : On ne peut être à la fois connecté au VPN (p) et au Wi-Fi invité (q).

Prémisse mineure : Or je suis connecté au VPN (p).

Conclusion ?

Solution : Je ne suis pas sur le Wi-Fi invité (¬q).

Explication : Ponendo Tollens pour incompatibilité - valide. Si p est vrai alors q doit être faux (et vice versa).

Exercice I-2

Énoncé : Impossible d'avoir la clé privée (p) et la partager en clair (q).

Prémisse mineure : Or je ne partage pas la clé (¬q).

Puis-je affirmer ¬p ?

Solution : Non.

Explication : Mineure négative → forme invalide. Ne pas partager la clé (¬q) ne permet pas de conclure sur p (on pourrait avoir la clé privée sans la partager).

Exercice I-3

Énoncé : On ne peut compiler et exécuter simultanément le même binaire.

Prémisse mineure : Or on exécute le binaire (q).

Conclusion ?

Solution : On ne le compile pas (¬p).

Explication : Valide (p = compiler, q = exécuter). Ponendo Tollens avec mineure affirmative : si q est vrai alors p doit être faux.

Exercice I-4

Énoncé : Une application ne doit pas être en "lecture seule" (p) et en "écriture" (q) en même temps.

Prémisse mineure : Elle n'est pas en lecture seule (¬p).

Peut-on conclure ¬q ?

Solution : Non.

Explication : Même piège que I-2. Mineure négative → forme invalide. Ne pas être en lecture seule (¬p) ne permet pas de conclure sur q.

Exercice I-5

Énoncé : On ne peut être loggé comme root (p) et utilisateur standard (q) simultanément.

Prémisse mineure : Je suis utilisateur standard (q).

Conclusion ?

Solution : Je ne suis pas root (¬p).

Explication : Ponendo Tollens valide. Mineure affirmative (q) permet de conclure ¬p.
